# 安全事件报告 (Postmortems)

> 安全事件分析和教训总结

---

## 概述

本目录记录 DeP2P 发生的安全事件及其事后分析，用于持续改进安全实践。

---

## 事件分类

| 级别 | 定义 | 示例 |
|------|------|------|
| P0 - 严重 | 数据泄露/服务完全不可用 | 私钥泄露 |
| P1 - 高 | 重大功能受损 | 认证绕过 |
| P2 - 中 | 部分功能受影响 | DoS 攻击成功 |
| P3 - 低 | 轻微影响 | 信息泄露（非敏感） |

## 事件响应流程

### 1. 发现

- 监控告警
- 用户报告
- 安全扫描
- 外部披露

### 2. 评估

- 确认问题
- 评估严重性
- 确定影响范围
- 通知相关人员

### 3. 响应

- 实施临时缓解
- 开发永久修复
- 测试修复
- 发布更新

### 4. 沟通

- 内部通报
- 用户通知（如需要）
- 安全公告（如需要）

### 5. 复盘

- 编写事后分析
- 识别改进项
- 更新流程
- 归档记录
