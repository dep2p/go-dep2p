## 概述

DeP2P 遵循语义化版本（Semantic Versioning 2.0.0），确保版本号清晰传达变更的性质和兼容性影响。

---

## 版本格式

```
v{MAJOR}.{MINOR}.{PATCH}[-{prerelease}][+{build}]

示例：
v1.0.0          正式版本
v1.1.0          新功能版本
v1.1.1          修复版本
v2.0.0-alpha.1  预发布版本
v2.0.0-rc.1     候选版本
```

---

## 版本变更规则

| 变更类型 | 版本升级 | 示例 |
|----------|----------|------|
| 破坏性变更 | MAJOR | v1.x.x → v2.0.0 |
| 新功能（兼容） | MINOR | v1.0.x → v1.1.0 |
| Bug 修复 | PATCH | v1.0.0 → v1.0.1 |
| 预发布 | prerelease | v2.0.0-alpha.1 |

---

## 发布渠道

| 渠道 | 分支 | 稳定性 | 用途 |
|------|------|:------:|------|
| `stable` | main | ⭐⭐⭐ | 生产使用 |
| `rc` | release/* | ⭐⭐ | 发布候选 |
| `alpha` | develop | ⭐ | 早期测试 |
