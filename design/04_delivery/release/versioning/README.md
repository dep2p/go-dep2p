# 版本策略

> 语义化版本规范

---

## 版本格式

```
v{MAJOR}.{MINOR}.{PATCH}[-{prerelease}]

示例：
v0.1.0-alpha     Alpha 预览版
v0.2.0-beta.1    Beta 测试版
v1.0.0-rc.1      候选版本
v1.0.0           正式版本
```

---

## 版本升级规则

| 变更类型 | 版本升级 | 示例 |
|----------|----------|------|
| 破坏性变更 | MAJOR | v1.x.x → v2.0.0 |
| 新功能（兼容） | MINOR | v1.0.x → v1.1.0 |
| Bug 修复 | PATCH | v1.0.0 → v1.0.1 |

---

## 预发布阶段

```
开发 → alpha → beta → rc → 正式版
```

| 阶段 | 稳定性 | 说明 |
|------|--------|------|
| alpha | ⭐ | 功能开发中，API 可能变化 |
| beta | ⭐⭐ | 功能完成，API 基本稳定 |
| rc | ⭐⭐⭐ | 候选版本，准备发布 |
