# core_security 需求

> 需求追溯与规格说明

---

## 功能需求

### FR-SEC-001: TLS 握手

| 属性 | 说明 |
|------|------|
| 描述 | 系统应能完成 TLS 1.3 握手 |
| 优先级 | P0 |
| 验收标准 | 成功建立加密通道 |

### FR-SEC-002: 证书生成

| 属性 | 说明 |
|------|------|
| 描述 | 系统应能生成自签名证书 |
| 优先级 | P0 |
| 验收标准 | 证书包含节点公钥 |

### FR-SEC-003: 身份验证

| 属性 | 说明 |
|------|------|
| 描述 | 系统应验证对端身份 (INV-001) |
| 优先级 | P0 |
| 验收标准 | RemotePeer == ExpectedPeer |

### FR-SEC-004: 公钥提取

| 属性 | 说明 |
|------|------|
| 描述 | 系统应能从证书提取公钥 |
| 优先级 | P0 |
| 验收标准 | 正确提取 Ed25519 公钥 |

---

## 非功能需求

### NFR-SEC-001: 加密强度

| 属性 | 说明 |
|------|------|
| 描述 | 使用 TLS 1.3，前向保密 |
| 验收标准 | 仅支持安全密码套件 |

### NFR-SEC-002: 性能

| 属性 | 说明 |
|------|------|
| 描述 | 握手应在 50ms 内完成（本地） |
| 验收标准 | 基准测试通过 |

---

## 需求追溯

| 需求 | 来源 | 说明 |
|------|------|------|
| FR-SEC-003 | INV-001 | 身份第一性不变量 |
| NFR-SEC-001 | 安全最佳实践 | 行业标准 |
