# core_realm 需求

> 需求追溯与规格说明

---

## 功能需求

### FR-RLM-001: 加入 Realm

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持使用 PSK 加入 Realm |
| 优先级 | P0 |
| 验收标准 | 正确派生 RealmID，注册到 Rendezvous |

### FR-RLM-002: 成员认证 (INV-002)

| 属性 | 说明 |
|------|------|
| 描述 | 系统应验证 Realm 成员资格 |
| 优先级 | P0 |
| 验收标准 | 非成员无法使用业务 API |

### FR-RLM-003: 成员发现

| 属性 | 说明 |
|------|------|
| 描述 | 系统应能发现同 Realm 成员 |
| 优先级 | P0 |
| 验收标准 | 通过 Rendezvous 发现成员 |

### FR-RLM-004: 离开 Realm

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持离开当前 Realm |
| 优先级 | P0 |
| 验收标准 | 清理资源，注销 Rendezvous |

### FR-RLM-005: 服务入口

| 属性 | 说明 |
|------|------|
| 描述 | Realm 应提供业务服务入口 |
| 优先级 | P0 |
| 验收标准 | 可获取 Messaging、PubSub、Streams |

---

## 非功能需求

### NFR-RLM-001: 认证性能

| 属性 | 说明 |
|------|------|
| 描述 | 认证应在 100ms 内完成 |
| 验收标准 | 基准测试通过 |
