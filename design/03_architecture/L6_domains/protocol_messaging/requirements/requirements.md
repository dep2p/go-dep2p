# core_messaging 需求

> 需求追溯与规格说明

---

## 功能需求

### FR-MSG-001: 请求-响应

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持请求-响应消息模式 |
| 优先级 | P0 |
| 验收标准 | 发送请求，等待并返回响应 |

### FR-MSG-002: 单向通知

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持单向消息发送 |
| 优先级 | P0 |
| 验收标准 | 发送消息，不等待响应 |

### FR-MSG-003: 发布订阅

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持主题发布订阅 |
| 优先级 | P0 |
| 验收标准 | 发布消息，所有订阅者收到 |

### FR-MSG-004: 消息验证

| 属性 | 说明 |
|------|------|
| 描述 | PubSub 应验证消息来源 |
| 优先级 | P0 |
| 验收标准 | 拒绝无效消息 |

### FR-MSG-005: 双向流

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持打开双向数据流 |
| 优先级 | P0 |
| 验收标准 | 可读写的流连接 |

---

## 非功能需求

### NFR-MSG-001: 消息延迟

| 属性 | 说明 |
|------|------|
| 描述 | 消息传递延迟 < 100ms（同局域网） |
| 验收标准 | 基准测试通过 |

### NFR-MSG-002: 吞吐量

| 属性 | 说明 |
|------|------|
| 描述 | PubSub 支持 1000 msg/s |
| 验收标准 | 压力测试通过 |
