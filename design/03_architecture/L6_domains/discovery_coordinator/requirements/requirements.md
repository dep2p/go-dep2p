# core_discovery 需求

> 需求追溯与规格说明

---

## 功能需求

### FR-DIS-001: DHT 发现

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持通过 Kademlia DHT 发现节点 |
| 优先级 | P0 |
| 验收标准 | 可发现网络中的节点 |

### FR-DIS-002: mDNS 发现

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持局域网内 mDNS 发现 |
| 优先级 | P1 |
| 验收标准 | 可发现同局域网节点 |

### FR-DIS-003: Rendezvous 发现

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持命名空间发现 |
| 优先级 | P0 |
| 验收标准 | 可按命名空间发现节点 |

### FR-DIS-004: 公告

| 属性 | 说明 |
|------|------|
| 描述 | 系统应能公告自身存在 |
| 优先级 | P0 |
| 验收标准 | 其他节点可发现本节点 |

---

## 非功能需求

### NFR-DIS-001: 发现延迟

| 属性 | 说明 |
|------|------|
| 描述 | mDNS 发现应在 5 秒内完成 |
| 验收标准 | 基准测试通过 |
