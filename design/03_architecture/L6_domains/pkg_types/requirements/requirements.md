# pkg_types 需求

> 需求追溯与规格说明

---

## 功能需求

### FR-TYP-001: NodeID 类型

| 属性 | 说明 |
|------|------|
| 描述 | 系统应提供 32 字节的 NodeID 类型 |
| 优先级 | P0 |
| 验收标准 | 支持比较、哈希、字符串转换 |

### FR-TYP-002: RealmID 类型

| 属性 | 说明 |
|------|------|
| 描述 | 系统应提供 32 字节的 RealmID 类型 |
| 优先级 | P0 |
| 验收标准 | 从 PSK 派生 |

### FR-TYP-003: PSK 类型

| 属性 | 说明 |
|------|------|
| 描述 | 系统应提供 32 字节的 PSK 类型 |
| 优先级 | P0 |
| 验收标准 | 不可打印到日志 |

### FR-TYP-004: Multiaddr 支持

| 属性 | 说明 |
|------|------|
| 描述 | 系统应支持 Multiaddr 地址格式 |
| 优先级 | P0 |
| 验收标准 | 正确解析和格式化 |

---

## 非功能需求

### NFR-TYP-001: 向后兼容

| 属性 | 说明 |
|------|------|
| 描述 | 类型变更需保持向后兼容 |
| 验收标准 | 不破坏现有代码 |
