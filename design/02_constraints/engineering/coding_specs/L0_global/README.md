## 核心原则

### 可读性优先

```
┌─────────────────────────────────────────┐
│              可读性原则                  │
├─────────────────────────────────────────┤
│                                         │
│  清晰 > 简洁 > 聪明                     │
│                                         │
│  • 代码是给人读的                       │
│  • 避免过度技巧                         │
│  • 显式优于隐式                         │
│                                         │
└─────────────────────────────────────────┘
```

### 一致性要求

| 方面 | 要求 |
|------|------|
| 格式化 | 统一使用工具格式化 |
| 命名 | 遵循统一命名规则 |
| 错误 | 统一错误处理模式 |
| 日志 | 统一日志格式和级别 |

---

## 应用范围

L0 规范适用于：

- 所有 Go 源代码文件
- 所有包和模块
- 所有层级的组件
