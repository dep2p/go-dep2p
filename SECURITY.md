# 安全策略

## 支持的版本

| 版本 | 支持状态          |
|------|-------------------|
| 1.0.x | :white_check_mark: |
| < 1.0 | :x:                |

## 报告漏洞

如果您发现安全漏洞，请**不要**公开报告。请通过以下方式私下报告：

1. **创建私有安全咨询**（推荐）：
   - 访问 [GitHub Security Advisory](https://github.com/dep2p/go-dep2p/security/advisories/new)
   - 创建新的安全咨询
   - 详细描述漏洞和复现步骤

2. **通过 GitHub Issues**：
   - 创建 Issue 并标记为敏感/安全问题
   - 我们会尽快处理并转为私有讨论

## 漏洞处理流程

1. **确认**：我们会在 48 小时内确认收到您的报告
2. **评估**：我们会在 7 天内评估漏洞的严重性
3. **修复**：我们会尽快修复漏洞
4. **披露**：修复后，我们会公开披露漏洞（在您同意的情况下）

## 安全最佳实践

使用 DeP2P 时，请遵循以下安全最佳实践：

- ✅ **密钥管理**：永远不要将私钥或 Realm Key 提交到代码仓库
- ✅ **TLS/QUIC**：DeP2P 默认使用 QUIC 和 TLS 1.3 加密连接
- ✅ **Realm 隔离**：使用不同的 Realm Key 隔离不同业务网络
- ✅ **输入验证**：验证所有用户输入和 NodeID 格式
- ✅ **依赖更新**：定期更新依赖包以获取安全补丁
- ✅ **最小权限**：使用最小必要权限运行应用
- ✅ **网络隔离**：生产环境使用独立的 Realm，避免与测试网络混用
