# 贡献指南

感谢您对 DeP2P 的兴趣！我们欢迎各种形式的贡献。

---

## 🎯 贡献方式

- **💻 代码贡献**：修复 Bug、实现新功能、优化性能
- **📚 文档贡献**：修复错误、添加说明、翻译文档、添加示例
- **🐛 Issue 报告**：报告 Bug、建议新功能、讨论设计问题
- **👀 代码审查**：审查 Pull Request、提供建设性反馈
- **💬 社区支持**：回答问题、分享使用经验

## 🚀 快速开始

1. **Fork 仓库**：Fork [DeP2P 仓库](https://github.com/dep2p/go-dep2p)
2. **创建分支**：从 `main` 分支创建功能分支
3. **提交更改**：提交您的更改并推送到您的 Fork
4. **创建 PR**：创建 Pull Request 并描述您的更改

---

## 📋 贡献前检查清单

- [ ] 代码遵循项目代码规范（见 [代码规范](docs/zh/contributing/code-style.md)）
- [ ] 添加了必要的测试
- [ ] 更新了相关文档
- [ ] 提交信息清晰明确
- [ ] 已阅读并同意 [行为准则](CODE_OF_CONDUCT.md)
